https://www.youtube.com/watch?v=sZOFEEhR4QY **Introduction** : La présentation porte sur l'IA agentique, un sujet crucial dans le domaine de l'intelligence artificielle. Julien Dubois, responsable des développeurs relation pour Java chez Microsoft et créateur de JHipster, explore trois concepts essentiels pour construire un agent IA : les structures d'output, le function calling et le modèle contexte protocole (MCP). Ces concepts, bien que récents, sont fondamentaux pour comprendre et utiliser l'IA de manière efficace. **Objectifs** : Les objectifs de la présentation sont de démystifier les concepts de structures d'output, de function calling et de modèle contexte protocole, de montrer comment ils fonctionnent à travers des démonstrations en direct, et de démontrer leur utilité dans la création d'agents IA interactifs et autonomes. **Points Clés** : - **Structures d'Output** : Les structures d'output permettent de forcer un schéma JSON pour les réponses de l'IA, assurant ainsi une communication programmatique claire et précise. Cela est particulièrement utile pour les développeurs Java qui peuvent mapper du JSON sur des objets Java. - **Function Calling** : Le function calling permet à l'IA d'exécuter des fonctions Java, lui donnant la capacité d'agir de manière autonome. Cela inclut la récupération d'informations et l'exécution d'actions spécifiques. - **Modèle Contexte Protocole (MCP)** : Le MCP est un standard permettant la communication entre un LLM et un serveur. Il ouvre la voie à des actions plus puissantes et diversifiées, en permettant l'intégration de serveurs distants et l'exécution de tâches complexes. **Méthodologie** : La présentation utilise le framework LangChain 4G, un outil open source connu pour aider à créer des applications IA en Java. Le nouveau SDK d'OpenAI, récemment publié, est également utilisé pour les démonstrations. Les démos incluent l'utilisation de schémas JSON pour forcer des structures d'output, l'exécution de fonctions Java via function calling, et l'intégration de serveurs MCP pour des actions distantes. **Résultats** : Les résultats montrent que les structures d'output permettent une communication claire et précise avec l'IA. Le function calling permet à l'IA d'exécuter des actions de manière autonome, tandis que le modèle MCP ouvre la voie à des intégrations plus puissantes et diversifiées. Les démonstrations montrent comment ces concepts peuvent être utilisés pour créer des agents IA interactifs et autonomes. **Conclusion** : L'IA agentique, avec ses structures d'output, function calling et modèle MCP, représente une avancée significative dans le domaine de l'intelligence artificielle. Ces outils permettent de créer des agents capables de communiquer de manière programmatique, d'exécuter des actions de manière autonome et d'intégrer des serveurs distants pour des tâches complexes. Bien que ces technologies soient encore en développement, elles offrent déjà des possibilités impressionnantes pour les développeurs et les utilisateurs d'IA.
